Chapter 02

學習前 vs 學習後的 Prompt

BEFORE 學習前

台灣熱炒店場景，一個男上司拿筷子指著魚
對女下屬說話，女下屬生氣把魚砸在他臉上，
男的被砸飛撞牆壁，站起來說我只是想吃
巴沙魚。搞笑風格，15秒。

沒有分鏡、沒有鏡頭語言、沒有音效、沒有 @ 引用、台詞混在描述裡

VS

AFTER 學習後

台灣熱炒店夜間公司聚餐，暖黃燈光混雜霓虹，
手持攝影微晃感，寫實都市喜劇質感，

@圖片1 的男性角色為男上司，
@圖片2 的女性角色為女下屬，

0-3秒：Medium Shot 中景拍攝聚餐桌面，
  上司拿筷子指著紅燒魚——
  台詞（男上司，油膩）："魚是好魚，就是刺多"
  環境音鍋鏟翻炒聲和杯盤碰撞；

4-6秒：Close-Up 女下屬臉部，困惑→暴怒——
  台詞（女下屬，暴怒）："你他媽吃個魚也要
  機機掰掰！" Whip Pan 甩鏡魚盤砸向面門；

7-10秒：Wide Shot 全景，撞穿牆壁人形大洞，
  慢動作捕捉，音效巨大撞擊+牆壁崩塌碎裂；

11-15秒：Low Angle 仰拍，灰頭土臉站起——
  台詞（男上司，委屈）："我只是想吃巴沙魚…"

禁止：任何文字、字幕、LOGO、水印

差異總結

維度BeforeAfter
時間控制完全沒有0-3s / 4-6s / 7-10s / 11-15s
鏡頭語言不存在Medium Shot / Close-Up / Whip Pan / Wide Shot / Low Angle
角色引用「一個男上司」@圖片1 @圖片2 三視角設定圖
音效設計沒有鍋鏟翻炒 / 砸臉撞擊 / 牆壁崩塌碎裂
台詞格式混在描述裡台詞（角色，情緒）："內容"
風格總綱「搞笑風格」暖黃燈光 + 手持微晃 + 寫實都市喜劇
禁止項沒有禁止文字 / 字幕 / LOGO / 水印
從「寫作文」進化到「寫分鏡腳本」 — 這就是 Prompt Skill 帶來的最大改變。
Chapter 03

720p vs 1080p — 為什麼低解析度反而更好

用完全相同的 prompt + 同樣的角色參考圖，分別生成 720p 和 1080p。直接看影片比較：

720p 原始版（15 秒一鏡）— 效果最好

720p 1280×720 · 15s · Seedance 2.0 · comedy 模式

1080p 同 prompt 重生（15 秒）— 品質下降

1080p 1920×1080 · 15s · 同一個 prompt · 動作連貫性明顯下降

分段拍攝嘗試（各 10 秒 1080p）

第 1 段 聚餐 → 指魚 → 砸魚

第 2 段 撞牆 → 站起 → 結尾台詞

分段拍攝的問題：兩段之間角色外觀和場景會不一致，拼接後有明顯斷裂感。

實測結論

720p ✓

  • 動作連貫性佳
  • 角色一致性高
  • Prompt 遵循度好
  • 分鏡轉場自然

1080p ✗

  • 動作斷裂不連貫
  • 角色臉部飄移
  • 部分分鏡被忽略
  • AI 破綻更明顯

四個核心原因

1. 算力稀釋效應

1080p = 207 萬像素/幀 vs 720p = 92 萬像素/幀 → 差距 2.25 倍
同樣的推理預算被稀釋，每個像素分到的「注意力」更少。

2. 原生訓練解析度

模型在 720p 或更低解析度上訓練。1080p 靠超解析度模組硬拉，不是原生生成。

3. 恐怖谷效應

720p 的輕微模糊反而掩蓋 AI 破綻（手指畸變、臉部微抖）。1080p 全部放大暴露。

4. 複雜 Prompt 的天花板

4 個分鏡 + 2 段台詞 + 撞牆物理效果 → 720p 剛好能處理，1080p 力不從心。

Chapter 04

Threads 爬蟲 — 繞過限制的方法

問題背景：Threads 的三重封鎖

🚫

沒有公開 API

Meta 沒有提供任何官方 API

📄

空殼 HTML

HTTP 請求只拿到空的 <div>

⚡

JS 動態載入

所有內容都靠 JavaScript 渲染

Firecrawl scrape、curl、requests、BeautifulSoup 全部失敗 — 抓到的都是空殼。

解決方案：Playwright Headless Browser

不走 HTTP 請求 → 直接用無頭瀏覽器載入頁面 → 等 JS 跑完 → 從 DOM 提取

Playwright 啟動

核心技術

# 1. 啟動 headless Chromium
browser = playwright.chromium.launch(headless=True)

# 2. 偽裝 User-Agent（看起來像一般瀏覽器）
page = browser.new_page(user_agent="Mozilla/5.0 ... Chrome/131.0")

# 3. 載入頁面並等 JS 渲染完成
page.goto(url, wait_until="networkidle")  # ← 關鍵

# 4. 從 rendered DOM 提取
posts = page.query_selector_all("[data-pressable-container]")

# 5. 搜尋模式走 Google site:threads.net
page.goto(f"https://www.google.com/search?q=site:threads.net+{query}")

四種模式

python threads_scraper.py user zuck          # 抓用戶最近貼文
python threads_scraper.py post <url>         # 單篇含互動數
python threads_scraper.py search "AI 台灣"    # Google 索引搜尋
python threads_scraper.py info zuck          # 用戶基本資訊

還可以應用在哪裡？

適用場景

  • SPA / CSR 網站（React / Vue / Angular）
  • 社群平台公開資料（Instagram / Twitter/X）
  • JS 渲染才有內容的頁面（Notion 公開頁）
  • 反爬蟲但不需登入的網站

進階組合

  • + Firecrawl interact：點擊、填表、翻頁、登入
  • + Whisper 語音轉文字：影片字幕結構化
  • + 排程任務：每日自動抓取趨勢分析
  • + LLM 分析：抓完直接餵 Claude 摘要
已整合到每日流行話題日報（daily_trending_topics.py），結合 Firecrawl 的 9 大來源破 AI 同溫層。

重點回顧

Prompt Skill

Seedance prompt 從「寫作文」進化到「寫分鏡腳本」。時間戳 + 鏡頭語言 + @ 引用 + 音效 + 台詞格式 + 禁止項。

720p + 超解析度

720p 生成（品質最穩）→ Real-ESRGAN 後期拉到 4K/5K。全部免費，本機 GPU 運算。

Playwright Headless

突破 SPA 網站的萬能鑰匙。Threads / IG / 任何 JS 渲染頁面。搭配排程 = 自動化資料收集。

免費工具鏈

三者結合 = 免費的影片製作 + 研究工具鏈。Seedance → 超解析度 → 爬蟲 → 日報，全自動零成本。
