# 你要做的 3 步驟（其他我都弄好了）

---

## 🔸 步驟 1：到 GitHub 網頁改 repo 名（30 秒）

1. 打開 <https://github.com/haochunhungster/haochun.github.io/settings>
2. 最上面 **Repository name** 欄位
3. 把 `haochun.github.io` 改成 **`haochunhungster.github.io`**
4. 按 **Rename** 確認

**為什麼要改**：GitHub Pages 規則 —— 主站 repo 名字必須等於 `<你的username>.github.io`，不然網址會變 `https://haochunhungster.github.io/haochun.github.io/` 這種醜東西。

---

## 🔸 步驟 2：在這個資料夾雙擊「初始化.bat」（2 分鐘）

位置：`C:\Users\whatp\做好的資料\公開\初始化.bat`

執行後會問你 `(y/n)`，輸入 `y` 按 Enter。

### ⚠️ 第一次推送時會發生這件事

推到 GitHub 那一步（第 5 步）時，會**跳出瀏覽器要你登入 GitHub**。
- 在瀏覽器按 **Authorize Git Credential Manager**
- 回到命令列視窗，它會自動繼續
- 你的登入資訊會被記住，**以後不會再問**

---

## 🔸 步驟 3：到 GitHub 開 Pages 功能（1 分鐘）

1. 推完之後，到 <https://github.com/haochunhungster/haochunhungster.github.io/settings/pages>
2. **Source** 選 `Deploy from a branch`
3. **Branch** 下拉選 `main`，右邊資料夾選 `/ (root)`
4. 按 **Save**
5. 等 1–2 分鐘
6. 打開 <https://haochunhungster.github.io/> —— **完成！**

---

## 之後要更新網站怎麼辦？

**只做兩件事**：
1. 把新的 `.html` 檔案丟進這個資料夾（或改 `index.html` 加連結）
2. 雙擊 **`發佈.bat`** → 問你這次更新什麼 → 輸入簡短說明 → Enter

30 秒後網站就更新了。

---

## 如果出錯怎麼辦？

### 「git 沒有認出來 / command not found」
→ 重開命令列視窗，或重開電腦（PATH 沒刷新）

### 「推送失敗：403 / Authentication failed」
→ 表示瀏覽器那步沒成功。重跑 `初始化.bat`，這次確實在瀏覽器按授權

### 「repository not found」
→ 表示步驟 1 的改名沒做，或改錯名字

### 其他錯誤
→ 截圖給我，直接貼進對話框，我看錯誤訊息解
